# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Что это

Одностраничное свадебное приглашение (Никита и Лера, 13.09.2026, Уфа) на **Astro 7**
и **Tailwind 4**, статическая сборка без UI-фреймворков и без клиентских зависимостей.
Пакетный менеджер — **pnpm**. Весь пользовательский текст, комментарии и коммиты — на русском.

## Команды

```bash
pnpm dev       # локальный сервер с HMR → http://localhost:4321
pnpm build     # сборка в dist/
pnpm preview   # посмотреть собранное
pnpm check     # astro check — типы в .astro и .ts
```

Тестов и линтера нет; `pnpm check` — единственный автоматический контроль, и он же
гоняется в CI до деплоя.

**TypeScript зафиксирован на 6.x.** Нативный TS 7 не отдаёт программный API, на который
опирается `astro check`, — обновление до 7 сломает и локальную проверку, и CI.

## Деплой

`.github/workflows/deploy-pages.yml`: пуш в `main` → `pnpm install --frozen-lockfile` →
`pnpm check` → `pnpm build` → публикация `dist/` на GitHub Pages.
Домен `leranikita.ru` задан в `public/CNAME`; так как домен кастомный, сайт живёт в корне
и `base` в `astro.config.mjs` не нужен.

`site` в `astro.config.mjs` — источник абсолютных URL для canonical, Open Graph и sitemap.
Sitemap генерирует интеграция `@astrojs/sitemap` (`sitemap-index.xml`), на него же ссылается
`public/robots.txt` — правьте их вместе.

## Архитектура

**`src/data/wedding.ts` — единственный источник правды.** Дата, площадка, контакт RSVP,
тайминг, палитра дресс-кода, напитки, правила, ссылки на карты. Меняете контент —
начинайте отсюда, а не с компонента.

Время задаётся ровно тремя константами: `DATE_ISO`, `GATHERING_TIME`, `END_TIME`
(плюс `TZ`). Из них считается всё остальное — `START_ISO`/`END_ISO`, таймер, `.ics`,
schema.org, а также `DATE_LONG` («13 сентября 2026»), `DATE_SHORT`, `DATE_DOTTED`
и `DATE_WEEKDAY`, которые уходят в `<title>`, Open Graph, обложку, подвал, «Коротко
о главном» и текст RSVP-сообщения. Дату руками нигде больше не пишут — перенос
свадьбы должен быть правкой одной строки.

Даты разбираются из строки (`DATE_ISO.split('-')`), а не через `Date.getDate()`:
последнее зависит от часового пояса машины, которая собирает сайт, и на CI может
дать сдвиг на сутки.

**`src/components/*.astro` — по компоненту на секцию**, порядок сборки задан в
`src/pages/index.astro`. Секции самодостаточны: каждая тянет свои данные и свой скрипт,
поэтому секцию можно удалить из `index.astro`, ничего больше не трогая.

**`worker/` — приёмник RSVP-заявок**, отдельный воркер Cloudflare. Форма шлёт
туда ответ гостя, воркер пересылает его в Телеграм. Живёт вне сборки Astro
и деплоится своей командой (`wrangler deploy`), поэтому исключён из `tsconfig.json`
— иначе `astro check` спотыкается о типы Cloudflare, которых в проекте нет.

Токен бота **только** в секретах Cloudflare. В `src/data/wedding.ts` лежит лишь
`RSVP_ENDPOINT` — публичный адрес воркера. Любой ключ в коде сайта попадает
в собранные файлы и становится общедоступным.

**`src/scripts/*.ts` — по модулю на поведение**, каждый в форме `function initX() { ... }`
+ вызов в конце файла. Ранний `return`, если нужного элемента нет.
Подключаются через `<script>` внутри своего компонента — Astro сам бандлит, минифицирует
и дедуплицирует. Оборачивать в функцию обязательно: файл без `import`/`export` TS считает
глобальным скриптом, и одноимённые `const` в разных модулях дают `ts(2451)`.

**Оформление — утилиты Tailwind в разметке; `src/styles/` — только то, что утилитами
не выражается.**

`src/styles/theme.css` — **дизайн-система целиком**: палитра, шрифты, шкала кеглей
`--text-*`, межстрочные `--leading-*`, трекинг `--tracking-*`, шкала отступов по
золотому сечению `--spacing-*`, три точки перелома `--breakpoint-*`. Всё это Tailwind
превращает в утилиты (`text-lg`, `leading-normal`, `bg-moss`, `p-md`). Новые размеры
берите из шкалы, а не пишите произвольные rem: она затем и сведена — до неё было
50 кеглей, 8 межстрочных и 8 брейкпоинтов вразнобой.

Там же живут свои утилиты `@utility shell` и `@utility head`. Они не «CSS в обход
Tailwind»: `@utility` создаёт настоящую утилиту в слое utilities. Заголовок стоит
в двенадцати местах — расписывать его строкой из шести утилит двенадцать раз значило бы
снова размазать типографику по файлам.

**Порядок слоёв объявлен в `theme.css`: `theme → project → utilities`.** Весь собственный
CSS проекта подключается в `global.css` с `layer(project)` и потому слабее утилит. Это
обязательно: бесслойный CSS сильнее любого слоёного независимо от специфичности, и без
слоя правило вида `.head { margin }` перебивало бы утилиту `mb-lg` в разметке.

Порядок импортов внутри слоя — порядок каскада, переставлять нельзя.
`shared.css` — общее для нескольких секций (`.char-img`, `.link-btn`, `.sprig-inline`,
кейфреймы). Остальные файлы — только невыразимое утилитами: многослойные градиенты
фона, `@keyframes`, псевдоэлементы со сложной геометрией и **классы состояний, которые
навешивает JS** (`.field--invalid`, `.to-top.show`, `.map-copy.copied`).

**Мина Tailwind 4:** утилиты `translate-*`, `scale-*`, `rotate-*` компилируются
в отдельные CSS-свойства `translate`/`scale`/`rotate`, а **не** в `transform`. Скрипты
проекта пишут `style.transform` (`progress.ts`) и `style.translate` (`parallax.ts`) —
там, где значение должно попасть именно в `transform`, пишите его явно:
`[transform:scaleX(0)]`, иначе браузер перемножит два независимых свойства и элемент
молча исчезнет.

Preflight (сброс Tailwind) намеренно **отключён**: подключаются только
`tailwindcss/theme.css` и `tailwindcss/utilities.css`. У сайта свой сброс в `base.css`,
а preflight поверх него менял вёрстку в половине секций и давал +18px к высоте страницы.

**Шрифты — свои, не с Google.** `src/styles/fonts.css` объявляет `@font-face`,
файлы лежат в `public/fonts/`. Так с критического пути ушли два соединения
с чужими доменами (`fonts.googleapis.com` за CSS, потом `fonts.gstatic.com`
за файлами), и запросы к Google больше не уходят вовсе.

Файлы подрезаны под кириллицу, латиницу, цифры и знаки — 580 КБ стали 380 КБ.
Подрезано под **полные алфавиты**, а не под текст страницы: в форме RSVP гость
печатает своё имя и пожелание, там может встретиться любая буква. Пересобрать —
`python3 scripts/fonts.py`; после этого проверьте страницу на «квадратики».

Предзагружаются только три файла — те, что видно на первом экране. Предзагрузить
всё значит отобрать канал у обложки, а она и есть LCP-элемент.

**Картинки — WebP** (персонажи, нужна прозрачность) и **JPEG** (`og-image`,
его разбирают краулеры мессенджеров, а WebP они поддерживают неровно).
В PNG они весили 3.2 МБ, из них обложка — 1.4 МБ. Подробности и способ
замены — в `public/assets/README.md`.

**`src/components/Sprite.astro`** — inline SVG-спрайт, подключается один раз в начале
страницы; ботанические веточки рисуются через `<use href="#sprig-*">`.

## Соглашения

- Анимации появления: класс `.reveal` / `.stagger`, обрабатывает `src/scripts/reveal.ts`.
  `<noscript>`-стиль в `src/layouts/Base.astro` принудительно показывает всё без JS —
  добавили новый анимируемый селектор, продублируйте его туда же.
- `prefers-reduced-motion` уважается везде; новые анимации тоже должны его учитывать.
- Картинки в `public/assets/` (Funko-персонажи, PNG с прозрачным фоном) опциональны:
  `onerror` убирает блок, если файла нет, — вёрстка не должна ломаться.
- Декоративная графика всегда `aria-hidden`.
- Многострочный текст в компонентах переносится Prettier'ом; неразрывные пробелы —
  literal `&nbsp;` в разметке, ` ` в строках внутри `.ts`.
