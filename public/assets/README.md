# assets

Картинки сайта. Персонажи — Funko на **прозрачном фоне**, формат **WebP**.

| Файл | Где на сайте |
|------|--------------|
| `couple.webp` | Обложка — пара стоят рядом (правый низ на десктопе, под именами на мобиле) |
| `groom-wave.webp` | «Приглашение» — жених машет рукой |
| `bride-bouquet.webp` | «О нарядах» — невеста с букетом зелени |
| `og-image.jpg` | Превью при отправке ссылки в мессенджер, 1200×630 |

Персонажи подключены декоративно (`aria-hidden`), с лёгким «дыханием»
(отключается при `prefers-reduced-motion`). Если файл убрать — блок с
персонажем аккуратно исчезает, вёрстка не ломается.

## Почему WebP, а не PNG

В PNG эти же картинки весили 3.2 МБ на страницу, из них обложка — 1.4 МБ.
На быстром канале незаметно, но приглашение открывают с телефона из мессенджера,
и там это секунды ожидания. В WebP при q92 они занимают 283 КБ — на 89% меньше,
разница на глаз неразличима (RMSE 1.3–1.7 по видимым пикселям).

`og-image` оставлен в JPEG, а не переведён в WebP: превью разбирают краулеры
мессенджеров и соцсетей, а они поддерживают WebP неровно. Прозрачность там
не нужна, и JPEG ничего не теряет.

## Как заменить персонажа

Положите файл с тем же именем: прозрачный фон, вертикальная или квадратная
ориентация, ~800–1200 px по большей стороне. Из PNG в WebP:

```bash
python3 -c "from PIL import Image; Image.open('новый.png').save('couple.webp','WEBP',quality=92,method=6)"
```

**Размер персонажа на странице подбирается не на глаз:** у файлов разные
пропорции, и одинаковые числа в CSS дают человечков разного роста.
Как считать — в комментарии к `.char-img` в `src/styles/shared.css`.
